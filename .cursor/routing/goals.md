# Weekly goals (`04-goals.md`)

Work record for the current week — not repo mechanics. Skills read/write it on Monday planning and when priorities change.

## File shape

```markdown
# Week of YYYY-MM-DD

**Sprint:** …
**ADO goal:** …

## Our goal
…

## Success criteria
- [ ] …

**Stretch:**
- [ ] …

## Priority

1. **Theme** — keywords, PBI ids, deadlines
…

**Deferred:** …

## Notes
…
```

## § Priority

**One ordered list for the week** — source of truth for sort order. `01-todos.md` § **This week** follows this list top to bottom. Sprint-assigned (`PBI #…`) and personal lines share the same ordering — not grouped by board vs chores.

| Skill | Role |
|---|---|
| `/mondayblues` | Writes § Priority for the new `week_of` |
| `/dawn`, `/synctodos` | Resort **entire** **This week** to match § Priority |

Do **not** paste the paragraph above into `04-goals.md` — only the numbered list, **Deferred**, and week-specific content.

## This week pointer

After `/mondayblues`, set in `01-todos.md` **## This week**:

`**Goal:** [04-goals.md](04-goals.md) — {one-line summary}`

Schedule tags on list lines → [tags.md](./tags.md).
